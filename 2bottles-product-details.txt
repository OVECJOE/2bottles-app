2bottles Product Details

Product Summary
2bottles is a mobile-first web application designed to help people coordinate fair, low-friction in-person meetups. Instead of one person traveling much farther than the other, the product uses location-aware logic to suggest practical midpoint-style places where both participants can arrive with balanced effort.

The current experience focuses on two-person coordination, with an architecture designed to evolve toward multi-person planning, richer social discovery, and authenticated premium capabilities.

Problem It Solves
When two people decide to meet, common friction points include:
- Unequal travel burden
- Back-and-forth chats to decide where to meet
- Lack of live visibility once both are en route
- No structured memory of good meetup spots

2bottles addresses this by combining fair venue suggestions, real-time location updates, and guided session flows from invite to meetup completion.

Target Audience
Primary audience:
- Friends coordinating city meetups
- Partners planning to meet in the middle
- Colleagues scheduling quick in-person catchups

Secondary audience:
- Small social groups who need fairness-oriented destination suggestions
- Users who value convenience, safety, and reduced coordination overhead

Market profile:
- Mobile-first users in urban or semi-urban areas
- Users comfortable with map-based interfaces and location sharing
- Users who want fast planning without installing a native app first

Core User Journey
1. A host creates a meetup session.
2. A partner joins through a shareable invite link.
3. The app gathers location context and suggests suitable meetup spots.
4. Both parties agree on a venue.
5. Live tracking and chat help both users coordinate while traveling.
6. Session ends with optional spot save/rating/media actions.

Current Technical Architecture
Frontend Stack
- Framework: Lit (Web Components)
- Language: TypeScript (strict mode)
- Bundler/dev server: Vite
- Router: @vaadin/router
- Mapping: maplibre-gl
- P2P realtime: peerjs (WebRTC data channels)
- Indexed storage: idb-keyval
- PWA support: Service Worker + manifest + install flow

Application Structure
- src/components: screen-level and reusable UI components
- src/services: business and integration services (geocoding, sessions, websocket, p2p, spot, notifications)
- src/store: app/session/location/ui state stores
- src/api: typed API client and endpoint wrappers
- src/sw: production service worker modules
- src/types: shared domain types

Runtime Model
- SPA bootstrapped from src/main.ts
- Route-driven UI shell in app mode with lazy-loaded screens
- Session lifecycle state persisted via IndexedDB/local storage
- Realtime sync handled via hybrid model:
  - direct peer-to-peer data channel when available
  - websocket channel for server-mediated updates/fallback sync

Location and Suggestion Logic
- Geocoding and reverse geocoding with caching and cooldown controls
- Venue candidate collection and progressive filtering/scoring
- Fairness-aware travel evaluation (ETA and distance balancing)
- Route-metric refinement where available

Reliability and UX Protections
- Dynamic import failure recovery and controlled reload behavior
- Service worker asset-missing signaling to reduce stale chunk issues
- Geolocation error handling with cool-off/retry strategy
- Offline-resilient shell behavior for key screens

Product Scope Today
Implemented focus:
- Two-person coordination
- Session creation, invite, join, venue agreement, live tracking, chat
- Spot save flow with backend-oriented API service contract

Planned expansions:
- Premium plans and authenticated features
- Multi-person coordination beyond two users
- Group fairness logic for common destinations and walk-bonding flows
- Stronger map-native social place discovery
- Enhanced recommendation intelligence powered by behavior and rating signals

Why This Product Is Differentiated
2bottles is not just a generic chat or map app. Its core differentiator is fairness-first meetup coordination, where the product actively reduces social friction by balancing travel burden and structuring decision flow from invitation to arrival.

Intended Value Proposition
- Faster meetup decisions
- Less negotiation fatigue
- More equitable travel experience
- Better confidence during transit
- Better future suggestions from saved/rated spots

Technology Maturity Snapshot
- Strong typed frontend foundation with strict TypeScript
- Modular service architecture for maintainability
- Unit-tested service layer for critical workflows
- Production-ready service worker and build pipeline

This foundation supports immediate deployment while remaining extensible for premium and multi-user roadmap capabilities.
